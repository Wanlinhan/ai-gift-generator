AI 图片礼物生成器需求与实现说明

一、项目定位

本项目是一个可本地部署的 AI 图片礼物生成器。使用者输入礼物主题提示词后，网站通过后端接口调用火山引擎 Ark Seedream 5.0 图片生成模型，按用户 prompt 描述生成礼物图片，并在页面中展示结果。

项目重点不是公开 SaaS 平台，而是“别人可以 clone 项目后在自己的电脑本地运行”。使用者需要使用自己的火山引擎 Ark API Key。

二、已完成功能

1. 前端页面

- 首页提供提示词输入框。
- 首页提供画幅选择：自动、1:1、16:9、9:16、4:3、3:4。
- 页面保留橙色品牌色、浅灰背景、白色圆角卡片和双栏布局。
- 左侧是生成参数和统计信息。
- 右侧是状态卡片和图片预览区域。
- 默认提示词为黑暗主题、金色点缀、游戏道具、透明质感、二次元风格。

2. 图片生成

- 用户点击“开始生成”后，前端请求本项目后端接口 /api/generate。
- 后端读取环境变量 ARK_API_KEY。
- 后端调用火山引擎 Ark 图片生成接口 /api/v3/images/generations。
- 默认模型为 doubao-seedream-5-0-260128。
- 用户 prompt 决定画面内容。
- 画幅控件决定是否向 Ark 传入 size；选择“自动”时不传 size，选择具体比例时传对应尺寸。
- 默认不添加水印。
- 生成成功后返回图片 URL。

3. 状态展示

当前状态包括：

- READY：页面初始状态
- GENERATING：图片正在生成
- SUCCESS：图片生成成功
- FAIL：图片生成失败

4. 结果展示

- 生成成功后在右侧预览区域展示图片。
- 用户可以点击“打开图片”在新标签页打开生成结果。

5. 本地部署

项目支持两种运行方式：

- Docker 运行：适合给别人使用，只需要 Docker Desktop 和自己的 API Key。
- Node.js 开发运行：适合开发调试。

三、暂不包含的功能

当前版本不包含：

- 登录系统
- 支付系统
- 生成历史
- 数据库
- 云端对象存储
- 多用户额度管理
- 公开接口鉴权
- 视频生成
- 异步任务轮询

这些功能可以作为后续版本扩展。

四、技术方案

技术栈：

- Next.js 15
- React 19
- TypeScript
- lucide-react
- Docker
- Docker Compose
- 火山引擎 Ark Seedream 5.0

主要文件：

- src/app/page.tsx：前端主页面
- src/app/globals.css：页面样式
- src/app/layout.tsx：页面 metadata 和根布局
- src/app/api/generate/route.ts：图片生成 API 路由
- src/lib/ark.ts：火山引擎 Ark 图片生成封装
- .env.example：环境变量示例
- Dockerfile：生产镜像构建
- docker-compose.yml：本地容器启动
- README.md：GitHub 使用说明

五、接口设计

POST /api/generate

请求示例：

{
  "prompt": "黑暗主题、金色点缀、透明质感的游戏道具摆件",
  "size": "2560x1440",
  "watermark": false
}

size 为可选字段。前端选择“自动”时不传 size；选择 16:9、9:16 等具体画幅时传对应尺寸。

成功响应：

{
  "status": "SUCCESS",
  "resultUrl": "https://..."
}

失败响应：

{
  "error": "错误信息"
}

六、安全要求

API Key 不能写入前端代码，不能提交到 GitHub。

正确调用链路：

浏览器页面 -> Next.js 后端接口 /api/generate -> 火山引擎 Ark API

环境变量：

ARK_API_KEY=your_volcengine_ark_api_key
ARK_BASE_URL=https://ark.cn-beijing.volces.com
ARK_MODEL=doubao-seedream-5-0-260128

.env 和 .env.local 已写入 .gitignore。

七、别人如何运行

Docker 方式：

git clone https://github.com/Wanlinhan/ai-gift-generator.git
cd ai-gift-generator
cp .env.example .env
docker compose up --build

然后访问：

http://localhost:3000

运行前需要在 .env 中填写自己的 ARK_API_KEY。

开发方式：

git clone https://github.com/Wanlinhan/ai-gift-generator.git
cd ai-gift-generator
npm install
cp .env.example .env.local
npm run dev

然后访问：

http://localhost:3000

八、验收标准

- 首页可以正常打开。
- 未配置 ARK_API_KEY 时，后端返回明确错误提示。
- 配置正确 API Key 且模型已开通时，可以生成图片。
- 浏览器源码中不包含真实 API Key。
- GitHub 仓库中不包含 .env 或 .env.local。
- Docker 运行方式写入 README。
- README 使用真实仓库地址 https://github.com/Wanlinhan/ai-gift-generator。
